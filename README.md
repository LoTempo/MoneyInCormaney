# Family Budget

Family Budget — учебное веб-приложение для самостоятельного ведения личных и семейных финансов. Оно помогает учитывать доходы и расходы, управлять сбережениями, задавать месячный лимит и анализировать финансовую статистику.

Проект распространяется как исходный код для самостоятельного запуска. Пользователь сам выбирает, где будут работать приложение и PostgreSQL и где будут храниться данные его экземпляра.

## Возможности

- регистрация и вход в аккаунт;
- независимое ведение личного бюджета без обязательного создания семьи;
- создание семьи и присоединение к ней по коду приглашения;
- раздельный учёт личных и семейных доходов и расходов;
- включение и отключение личного или семейного бюджета в настройках;
- личные и семейные категории с подкатегориями и собственными цветами;
- отдельное управление личными и семейными сбережениями;
- ручная настройка доступного месячного лимита расходов;
- аналитика доходов и расходов по месяцам, годам и категориям;
- отдельная история изменения сбережений;
- защита операций других участников семьи от изменения и удаления;
- выход из семьи, удаление участников и передача прав владельца;
- настройка профиля, валюты, адреса электронной почты и телефона;
- адаптивный интерфейс для компьютеров и мобильных устройств.

Личные элементы интерфейса выделены синими оттенками, а семейные — фиолетовыми. Это помогает сразу понимать, к какому бюджету относится операция или показатель.

## Полностью локальный запуск в Windows

При таком варианте и Python-приложение, и база данных PostgreSQL работают на вашем компьютере. Для запуска понадобятся:

- Git;
- Python 3.12 или новее;
- PostgreSQL 15 или новее;
- терминал Windows PowerShell.

### 1. Загрузка основного кода

Клонируйте репозиторий и перейдите в созданную папку:

```powershell
git clone <адрес-вашего-репозитория>
cd <папка-репозитория>
```

Вместо клонирования проект можно скачать с GitHub как ZIP-архив и распаковать. В этом случае команда `git clone` не нужна.

### 2. Подготовка Python

Создайте виртуальное окружение:

```powershell
python -m venv .venv
```

Активируйте его:

```powershell
.\.venv\Scripts\Activate.ps1
```

Установите библиотеки проекта:

```powershell
python -m pip install -r requirements.txt
```

Виртуальное окружение изолирует библиотеки Family Budget от остальных Python-проектов на компьютере.

### 3. Создание локальной базы PostgreSQL

Установите PostgreSQL, если он ещё не установлен. Во время установки запомните пароль главного пользователя `postgres`.

Базу и отдельного пользователя можно создать через графическую программу pgAdmin или через терминал `psql`. Для варианта с `psql` выполните:

```powershell
psql -U postgres
```

В открывшейся консоли PostgreSQL выполните команды:

```sql
CREATE USER family_budget_user WITH PASSWORD 'придумайте_надежный_пароль';
CREATE DATABASE family_budget OWNER family_budget_user;
\q
```

Первая команда создаёт пользователя базы. Вторая создаёт пустую базу `family_budget` и назначает этого пользователя её владельцем. `\q` закрывает консоль PostgreSQL.

### 4. Настройка `.env`

Создайте рабочий файл настроек из примера:

```powershell
Copy-Item .env.example .env
```

Откройте `.env` в текстовом редакторе. Для локальной базы оставьте строки `DATABASE_URL` и `DATABASE_ADMIN_URL` пустыми, а ниже укажите локальные параметры:

```text
DATABASE_URL=
DATABASE_ADMIN_URL=
DATABASE_HOST=localhost
DATABASE_PORT=5432
DATABASE_NAME=family_budget
DATABASE_USER=family_budget_user
DATABASE_PASSWORD=пароль_из_предыдущего_шага
```

Также замените `SECRET_KEY` на длинную случайную строку. Создать её можно командой:

```powershell
python -c "import secrets; print(secrets.token_urlsafe(48))"
```

Скопируйте полученную строку в `SECRET_KEY`. Файл `.env` содержит пароли и не должен попадать в Git.

### 5. Развёртывание SQL-структуры

Убедитесь, что PostgreSQL запущен, а затем выполните:

```powershell
python -m flask --app run init-db
```

Команда подключится к базе из `.env` и выполнит файл `sql/schema.sql`. В пустой базе появятся таблицы, ограничения и индексы, необходимые приложению. Демонстрационные пользовательские записи команда не создаёт.

Команда предназначена для первоначальной подготовки пустой базы. Не запускайте её без необходимости для базы с важными рабочими данными.

### 6. Запуск сайта

Запустите сервер:

```powershell
python run.py
```

После этого откройте в браузере:

```text
http://127.0.0.1:5000
```

Пока окно PowerShell с сервером открыто, сайт работает. Для остановки нажмите `Ctrl+C`.

При следующем запуске не нужно повторно создавать базу и устанавливать библиотеки. Достаточно открыть папку проекта, активировать `.venv` и выполнить `python run.py`.

## Варианты размещения

Приложение и база данных не обязаны находиться на одном компьютере.

### Где может находиться PostgreSQL

- на том же локальном компьютере, где запущен Python;
- на собственном VPS с установленным PostgreSQL;
- в облачном сервисе управляемых баз данных, например Neon или аналогичном.

Для внешней или облачной базы провайдер обычно выдаёт полную строку подключения. Её нужно сохранить в `DATABASE_URL`:

```text
DATABASE_URL=postgresql://USER:PASSWORD@HOST:5432/DATABASE?sslmode=require
```

Если `DATABASE_URL` заполнен, приложение использует его и не использует отдельные `DATABASE_HOST`, `DATABASE_PORT`, `DATABASE_NAME`, `DATABASE_USER` и `DATABASE_PASSWORD`.

Некоторые провайдеры выдают два адреса: pooled-адрес для обычной работы и direct/admin-адрес для создания таблиц. В таком случае pooled-адрес указывается в `DATABASE_URL`, а direct/admin-адрес — в `DATABASE_ADMIN_URL`. Если провайдер выдаёт только один адрес, `DATABASE_ADMIN_URL` можно оставить пустым.

После подключения новой пустой внешней базы один раз выполните `python -m flask --app run init-db`, чтобы загрузить в неё структуру из `sql/schema.sql`.

### Где может находиться сайт

Python-приложение можно запустить:

- на своём компьютере для локального использования;
- на собственном VPS;
- на Python-хостинге, например Alwaysdata или аналогичном;
- в контейнерном сервисе через находящийся в проекте `Dockerfile`.

На хостинге необходимо установить зависимости из `requirements.txt`, добавить переменные окружения и указать способ запуска приложения. WSGI-точка входа проекта находится в `wsgi.py` и называется `application`. Для рабочего режима следует задать как минимум:

```text
APP_ENV=production
FLASK_DEBUG=false
SECRET_KEY=длинный_случайный_секрет
DATABASE_URL=строка_подключения_к_PostgreSQL
```

Настоящие секреты нужно добавлять через панель переменных окружения хостинга, а не записывать в GitHub.

## Основные переменные окружения

| Переменная | Назначение |
| --- | --- |
| `APP_ENV` | Режим работы приложения: `development` или `production` |
| `SECRET_KEY` | Секретный ключ для защиты пользовательских сессий |
| `DATABASE_URL` | Необязательная полная строка подключения к локальному или внешнему PostgreSQL |
| `DATABASE_ADMIN_URL` | Необязательная отдельная строка для создания таблиц во внешней базе |
| `DATABASE_HOST` | Адрес PostgreSQL, если `DATABASE_URL` не задан |
| `DATABASE_PORT` | Порт PostgreSQL |
| `DATABASE_NAME` | Имя базы данных |
| `DATABASE_USER` | Пользователь базы данных |
| `DATABASE_PASSWORD` | Пароль пользователя базы данных |
| `FLASK_DEBUG` | Включение режима отладки при локальной разработке |
| `SERVER_HOST` | Адрес, на котором приложение принимает подключения |
| `SERVER_PORT` | Локальный порт приложения |

```text

├── static/       # стили, JavaScript и изображения
```

## Проверка проекта

Обычные автоматические проверки запускаются командой:

```powershell
python -m unittest discover -s tests -v
```

Интеграционные проверки с PostgreSQL по умолчанию отключены, чтобы случайно не изменить рабочие данные.

## Безопасность

Пароли пользователей сохраняются в виде криптографических хешей. Формы защищены CSRF-токенами, а сервер добавляет базовые HTTP-заголовки безопасности.

Никогда не публикуйте в репозитории файл `.env`, пароли базы данных, `SECRET_KEY` и другие секреты. Если секрет случайно попал в Git, одного удаления из файла недостаточно — его нужно немедленно заменить.
